## Трайдинговая платформа на Solidity для Ethereum

Контракт `Trading_Platform` реализует минималистичную торговую площадку формата заказчик ↔ поставщик. Решение позволяет размещать заказы, откликаться на них, выбирать поставщика, принимать товар и завершать сделку с выплатой вознаграждения, удерживая комиссию платформы.

### Структура репозитория
- `Trading_Platform` — основной смарт-контракт Solidity
- `Курсовой проект.pdf` — описание проекта

## Возможности смарт-контракта
- Регистрация заказов (`create_order`) с контактными данными покупателя.
- Подача откликов на заказ (`create_answer`) от поставщиков.
- Просмотр активных и всех заказов/ответов (различные `get_*` функции).
- Выбор подходящего предложения заказчиком (`choose_answer`) с оплатой и блокировкой средств.
- Подтверждение отправки товара поставщиком (`product_sent`) и завершение сделки заказчиком (`complete_order`).
- Отмена заказов и откликов до выбора контрагента (`delete_order`, `delete_answer`).
- Сбор статистики по пользователям и платформе (`get_tp_stats`, `get_user_stats`).

### Ключевые сущности
- `Status` — enum жизненного цикла заказа/ответа: `Posted → Chosen → Sent → Completed` (+ `Cancelled`).
- `Order` — заказ покупателя.
- `Answer` — предложение поставщика.
- `TPStats` — агрегированная статистика по заказам и ответам.
